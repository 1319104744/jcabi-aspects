Logging Exceptions Thrown out of Java Methods and Swallowing Them

Annotate your methods with @Quietly annotation and every exception thrown out of the method will not be propagated, but it will be logged through SLF4J:

public class Resource {
  private Object content;
  @Quietly
  public void load(URL url) {
    this.content = url.openConnection().getContent();
  }
}

If an exception occurs, it will not be thrown to the caller and it will be logged through SLF4J logging facility.

Also, take a look at @LogExceptions annotation, which does the same but it propagates exceptions thrown to the caller.

The mechanism is implemented with AOP/AspectJ. Read how to integrate it into your pom.xml.
